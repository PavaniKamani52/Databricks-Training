# Week 2 - Day 6: REGEX Foundation & SQL Data Quality Analysis

## Project Overview

This project explores the fundamentals of Regular Expressions (REGEX) in MySQL and Data Quality Analysis using SQL. It focuses on validating, cleaning, and analyzing raw business data through practical, real-world scenarios commonly encountered in data engineering and database management.

## Features

### REGEX Operations

* Pattern matching using `REGEXP`
* Email validation and verification
* Phone number validation
* Username and domain extraction
* String cleaning and transformation
* Text pattern analysis

### Data Quality Analysis

* NULL value detection
* Duplicate record identification
* Negative value validation
* Updated record tracking
* Data cleansing and preprocessing

## SQL Concepts Used

* `REGEXP`
* `REPLACE()`
* `SUBSTRING_INDEX()`
* `COUNT()`
* `GROUP BY`
* `HAVING`
* `IS NULL`

## Business Use Cases

* Customer data validation
* Contact information verification
* Data quality auditing
* ETL preprocessing workflows
* Raw data ingestion checks
* Duplicate record management
* Transaction data validation

## Learning Outcomes

By completing this project, I gained hands-on experience in:

* Implementing REGEX patterns in MySQL
* Validating emails and phone numbers
* Extracting meaningful information from text data
* Detecting and resolving data quality issues
* Handling NULL values and duplicates
* Understanding ETL preprocessing concepts
* Working with real-world dirty datasets

## Technologies Used

* MySQL 8+
* SQL
* Regular Expressions (REGEX)

## Project Objective

The primary goal of this project is to strengthen SQL and REGEX fundamentals while developing practical skills in data validation, data cleaning, and quality analysis. The project simulates common challenges faced in real-world database systems and data engineering pipelines.

**Skills Demonstrated:** SQL Querying, Data Cleaning, Data Validation, REGEX Pattern Matching, Data Quality Analysis, ETL Fundamentals.
