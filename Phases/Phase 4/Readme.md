# Phase 4 – Business Pipeline & Analytics using PySpark

## Overview

This project demonstrates an end-to-end **business analytics pipeline** using **PySpark** and **Apache Spark**. It covers data ingestion, cleaning, transformation, dataset integration, customer segmentation, and business reporting. Each analytical task is implemented using both **Spark SQL** and the **PySpark DataFrame API** to demonstrate their equivalent functionality.

---

## Objectives

* Build an end-to-end ETL and analytics pipeline.
* Clean and validate transactional data.
* Join customer and sales datasets.
* Generate meaningful business insights.
* Implement business queries using SQL and PySpark.
* Create a reusable reporting pipeline.

## Workflow

```text
        Data Ingestion
              │
     Read Customer & Sales Data
              │
        Data Cleaning
 ├─ Remove Null Values
 ├─ Remove Duplicates
 └─ Filter Invalid Records
              │
       Data Integration
      Join on customer_id
              │
      Business Analytics
 ├─ Daily Sales
 ├─ City-wise Revenue
 ├─ Top 5 Customers
 ├─ Repeat Customers
 ├─ Customer Segmentation
 └─ Final Reporting Table
              │
         Save Output
```

---

## Business Analytics

* **Daily Sales:** Calculate total sales for each day.
* **City-wise Revenue:** Compute revenue generated by each city.
* **Top 5 Customers:** Identify customers with the highest total spending.
* **Repeat Customers:** Find customers with more than one order.
* **Customer Segmentation:** Classify customers into **Gold**, **Silver**, and **Bronze** based on total spending.
* **Final Report:** Generate a consolidated report containing customer details, spending, order count, city, and segment.

---

## SQL to PySpark

Each business problem is implemented using:

* Spark SQL
* Equivalent PySpark DataFrame API

This demonstrates how SQL operations such as filtering, joins, aggregations, grouping, and conditional logic can be translated into scalable PySpark transformations.
